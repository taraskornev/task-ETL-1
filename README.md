# task-ETL-1

# Завдання для дата-аналітика (навчальний проєкт)

## Загальний опис  
Мета цього проєкту — опрацювати набір даних за допомогою бібліотеки Pandas: очистити його, виконати трансформації, створити нові колонки, відфільтрувати інформацію й зберегти фінальний результат у форматах, придатних для подальшого аналізу. Це навчальний проєкт, заснований на підходах, описаних у статті: *“Pandas: управління даними проєкту (2)”*. (Джерело: Medium)

---

## Мета проєкту  
- Отримати практичний досвід застосування Pandas для обробки реальних табличних даних.  
- Навчитись перетворювати сирі дані у структурований та аналітично готовий формат.  
- Підготувати DataFrame, який можна буде аналізувати або візуалізувати.

---

## Основні цілі  
1. Завантажити та дослідити вхідні дані.  
2. Видалити зайві рядки та колонки.  
3. Додати нові колонки на основі існуючих даних.  
4. Вставити нові записи (рядки) у DataFrame.  
5. Виконати фільтрацію даних за різними критеріями.  
6. Створити підвибірки (сегменти) даних за логічними умовами.  
7. Зберегти підготовлений DataFrame у зовнішні формати для подальшого використання.

---

## План реалізації

1. **Імпорт і попередній аналіз даних**  
   - Імпортувати Pandas та зчитати набір даних (наприклад, CSV).  
   - Зробити первинний огляд: `df.head()`, `df.info()`, `df.describe()`.  
   - Визначити, які колонки можуть бути непотрібними (зайві або зайві роки, “AREA” тощо).

2. **Очищення даних**  
   - Видалити непотрібні колонки за допомогою `drop()`.  
   - Перевірити, чи є пропущені значення, дублікати або рядки, які варто видалити.  
   - За потреби — видалити рядки з певними індексами або за мітками.

3. **Додавання нових колонок**  
   - Створити один або кілька нових стовпців. Наприклад, обчислити “total” або “difference” між числовими колонками.  
   - Використати `insert()` для розміщення нових колонок у бажаному порядку.

4. **Додавання нових рядків**  
   - Створити окремий DataFrame з новими рядками.  
   - Додати їх до основного DataFrame через `append()`.  
   - Переконатись, що індекси нових рядків коректні (наприклад, названі або числові).

5. **Фільтрація даних**  
   - Використати `loc[]` з логічними умовами для вибірки рядків, наприклад:  
     - за конкретним значенням у колонці;  
     - за текстовими умовами (наприклад, закінчення email чи інший шаблон);  
     - поєднувати умови через `&` або `|`.  
   - Використати `iloc[]` для позиційної вибірки (наприклад, перші / останні рядки, або специфічні позиції).

6. **Структурування та сегментація**  
   - Створити підвибірки даних (сегменти) на основі бізнес-логіки або аналітичних умов.  
   - Наприклад, вибрати користувачів, які задовольняють кільком умовам, або згрупувати дані за категоріями.

7. **Експорт фінального набору даних**  
   - Зберегти підготовлений DataFrame у CSV файл (`to_csv`).  
   - Зберегти також у Excel (`to_excel`).  
   - Переконатись, що форматування файлів відповідає стандартам: немає зайвих індексів, кодування UTF-8, читабельні назви колонок.

8. **Документування та звіт**  
   - Написати короткий “readme” чи нотатки (Markdown або Jupyter Notebook), де описано, що зроблено:  
     - які колонки видалено / додано;  
     - які фільтри застосовано;  
     - логіка додавання нових рядків;  
     - як експортуються дані.  
   - Пояснити, чому прийняті саме такі рішення (обґрунтування трансформацій).

---

## Критерії успіху / приймальні умови  
- DataFrame має бути без «шумових» колонок і непотрібних рядків.  
- Нові колонки мають коректні значення (логічні, обґрунтовані).  
- Фільтровані підвибірки правильно відображають умови, задані в задачі.  
- Експортовані файли (CSV / Excel) відкриваються і містять правильно структуровану інформацію.  
- Документ / ноутбук має достатню кількість коментарів / пояснень, щоб інший аналітик або менеджер міг зрозуміти, що й навіщо зроблено.

## Додаткові поради  
- При роботі з новим DataFrame роби часто `.copy()`, щоб не зіпсувати оригінал.  
- Використовуй версійний контроль файлів (наприклад, `git`), щоб можна було відкотитися до попередніх станів.  
- Перевіряй результати після кожного великого кроку: виводь `head()`, `info()`, `value_counts()` для категорійних колонок.  
- Пиши коментарі до коду — це корисно як для тебе, так і для інших, хто буде читати твою роботу.
